# Event Booking Microservices

Данный проект представляет собой микросервисную систему для бронирования мест на события.
Система реализована в рамках учебного задания и демонстрирует основные принципы микросервисной архитектуры: изоляцию сервисов, сетевое взаимодействие, контрактно-ориентированное API и контейнеризацию.

---

## Архитектура системы

Проект состоит из четырёх независимых микросервисов:

* **Account Service** — управление аккаунтами пользователей
* **Event Service** — управление событиями и доступными местами
* **Booking Service** — бронирование и отмена бронирований
* **Notification Service** — отправка уведомлений о бронированиях

Каждый сервис:

* является отдельным Go-приложением;
* работает в собственном Docker-контейнере;
* имеет собственную базу данных PostgreSQL (кроме Notification Service);
* взаимодействует с другими сервисами **только через gRPC**.

---

## Используемые технологии

* **Язык программирования:** Go (Golang)
* **Межсервисное взаимодействие:** gRPC
* **Описание контрактов:** Protocol Buffers
* **Базы данных:** PostgreSQL
* **Контейнеризация:** Docker
* **Оркестрация контейнеров:** Docker Compose

---

## Запуск проекта

### Запуск всей системы

Из корня проекта выполните команду:

```bash
docker-compose up -d
```

Все сервисы и базы данных будут запущены одной командой.

| Сервис               | Порт  |

| Account Service      | 50051 |
| Event Service        | 50052 |
| Booking Service      | 50053 |

PostgreSQL для каждого сервиса также запускается в отдельном контейнере.

## Генерация protobuf-кода

Для генерации Go-кода из `.proto` файлов:

```bash
protoc \
  --go_out=paths=source_relative:. \
  --go-grpc_out=paths=source_relative:. \
  proto/<service>/<file>.proto
```
